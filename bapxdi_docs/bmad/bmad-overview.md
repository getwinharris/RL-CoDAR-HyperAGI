WHAT BMAD IS:
Build More Architect Dreams — AI-driven agentic framework.
Orchestrates specialized AI agent personas through structured workflows.
Works with Claude Code, Cursor, Windsurf, GitHub Copilot, Roo.
Install: npx bmad-method install

CORE AGENTS:
Analyst — brainstorming, research, brief creation
PM — PRD creation, requirements
Architect — technical architecture, system design
Scrum Master — story sharding, sprint planning
Dev — implementation, coding, debugging
QA — testing, quality, edge cases
Orchestrator — coordinates all agents, answers bmad-help

SKILLS ARCHITECTURE (.claude/skills/):
Skills are SKILL.md files — structured instructions for specific capabilities.
Composable — multiple skills combine for complex tasks.
Same structure as bapXdi docs/ folder.

PARTY MODE:
Multiple agent personas in one session simultaneously.
Agents collaborate, debate, produce better outputs.
/party command activates.

MODULE STRUCTURE:
core — framework always installed
bmm — BMad Method software development
bmb — BMad Builder for custom agents
cis — Creative Intelligence Suite 150+ creative techniques
_cfg — your customizations that survive all updates

WHAT THIS MEANS FOR BAPXDI:
BMAD structured story files = byte documents that condition bapXdi generation.
Drop any BMAD PRD or Architecture doc into user_docs/ = instant model brain.
bapXdi reads BMAD documents as bytes — no tokenizer, no embedding, no training.
BMAD + bapXdi = structured human-readable documents that ARE the AI.
Every BMAD agent's output becomes bapXdi's next generation signal.
